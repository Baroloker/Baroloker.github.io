书写计划 0.1.0

说明

烂漫岂无意，为君占年华，连春日的更迭都尚有留恋，我为什么不能够在斗转变换中做点什么呢？身为计算机学子，深感时光蹉跎的愧怍。一方面来源于我自己没能够奋力找到适合自己精进的环境，另一方面来源于我没有琢炼到平和的心境，于是没有能够将所学能够极大限度地所用。不过既然选择了这条路，虽然没有任何天分和基础，我也宁愿苦求，留下些许足迹就好。万幸的是，兜兜转转，终于开启了自己博客撰写的第一站，希望能够借此机会发现学习过程中疏漏以及督促自己向前。

试想

我预计从以下内容进行我的学习的点滴记录：

  1. Coding Time 最熟悉的一栏？用来记录平时编码过程中的感悟，可能是算法或者项目的学习经历
  2. Mathematic Time 最费脑的一栏？用来检测平时的数学学习状况，毕竟真喜欢也真的重要谁能想到最初是喜欢数学才选择的计算机呢，虽然爆菜哈哈
  3. Research Time 最摆烂的一栏？估计会用来更新科研状态，虽然什么都不懂，也摸不清自己想要的东西——不过还是有一个计算机视觉和林草人工智能处理方向的项目，好事多磨吧
  4. Reading Time 最喜欢的一栏？毕竟读书是我除了杂学和足球能感到最快乐也是同样快乐的事情，记录思考是一件很酷的事情，因为她能够提醒和指导认知
  5. Language Time 最需要的一栏？英语作为当今世界最广泛流传的语言，不学肯定是不行的，况且新玩意儿很多都来自于计算机的发源地，避免“一条腿”走路
  6. Exploring Time 最特殊的一栏？希望能多挖点坑自己去填，我想无论是新技术、新项目、新思路的学习还是新观点、新思考、新认知的碰撞都应该囊括其中，也许学计算机就应该有这种意识，计算机和物理不同，几乎整个计算机世界都是人为制定的规则（除去少数物理、材料、自反规则限制），只要去学，总有一天能够有所获取，即使少之又少那也无妨
  7. Jotting Time 最繁杂的一栏？但是技多不压身，我一直觉得如果觉得没有方向，不如就多走、多做、多看、多学、多听、多想，就像DFS一样我都走过了，我还怕没有最终的答案吗？
  8. Leaving a Note 最留白的一栏？

近期

Linus 在 Just For Fun 一书中提到一开始程序员都是为了好玩有趣或者好看才做出了一些游戏、系统（界面好看这种原因）等项目，之后才意识到有些东西能商用和或者真有点其他使用的功能才进行正经的改进。我想我也应该好好学一下正经的好看的 Markdown 语法编辑了，以及需要先多做点 hexo 界面上选取和修改，可能是清明假期的主要任务吧：

  • 修改博客样式
  • 学习文档撰写，争取更新书写计划的版本
  • 没几天就要蓝桥杯了，寄
  • 拟定四月份书单和锻炼计划
  • 拟定四月计算机学习目标和数学学习目标
